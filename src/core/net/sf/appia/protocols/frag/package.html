This protocol fragments messages according to the lower layers maximum Protocol Data Unit (PDU) Size.

Messages are splited in the following way:

  • all but the last fragment are sent on FragEvent events
  • the last fragment is sent with the original event type
upon reception, all but the last fragment are concatenated in the last one, and this one is the event sent to channel.

Headers contains two fields:

  • a unique msgid
  • a fragment number

This protocol does not provide reliability. If a fragment is received out-of-order, the whole message is discarded. In order to provide reliable delivery, a reliable protocol (like FIFO) should be provided bellow the fragmentation protocol.
